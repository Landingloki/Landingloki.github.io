hexo+github page建站记录

“从这里记述这个小站的诞生。”

前言

如果不想听我唠叨就请直接移步正文吧。

这个博客是我一时兴起用了一个下午就完成的东西。本来我并没有打算搞一个这个，而是想了解一些关于VPN和Shadowsocks的知识，不过因为这俩太过专业，看得我头大，正在这时我发现一篇文章，介绍了非常简单的hexo+GitHubPages建站，让头大的我瞬间有了精神。

在看Shadowsocks的相关文章时进入了Huanhoo的个人blog，在那里我看到了她(应该是女性吧)的Huanhoo和小站的诞生这篇文章。然后

我就有了这个属于我的小站。

用这个方法建站确实非常简单，顺利的话can be finished in a few minutes，但是由于纠结于主题的选择和使用，我拖了好长时间，最后几乎一个下午的结果居然就是直接把最简朴的默认主题放了上去，TAT。当然我没有任何贬低默认主题的意思，只能说我现在毕竟还是刚刚开始用hexo，主题还是先放放吧。

正文

先前准备

为了这个站你所需要的东西有：

  • Node.js
  • Git
  • Hexo
  • 一个Markdown编辑器

以上的链接是直达对应的官网的。

对我而言，Node.js和Git早就待在我的PC上了，而Hexo也是利用安装了Node.js之后的npm包就可以轻松安装。而Markdown编辑器，我用的是大微软的VS Code，毕竟也是安装了好久了而且它可以预览你所写的东西，我觉得挺好的其实是懒得安装其他的了。

具体步骤

先建一个文件夹，文件名为 yourname.github.io，这是为了要和GitHub Page的仓库名对应。然后在命令行里进入这个文件夹，执行hexo init命令，完成后再执行hexo s(s 是 server 的缩写)，这之后就会在本地起一个服务器,把那个地址放到浏览器里就可以看到默认的hexo主题landscape了(没错，就是我现在用的)。

关于主题

接下来可以去Hexo的官网挑选一个博客主题，每个主题的配置方式都不一样，具体还是看主题的使用文档。顺带一提，我就是这里浪费了好多时间，其实应该以后慢慢研究的。

新建文章

执行hexo new + 文章名就可以新建一个文章了,具体路径是source\\_post\文章名.md。打开你的Markdown编辑器就可以写了，当然要先了解了解Markdown的语法，不过写一篇文章不就知道了吗。
